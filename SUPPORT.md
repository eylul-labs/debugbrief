# Support

DebugBrief is in early MVP development.

## Before Publishing

Use local project notes and direct owner feedback.

## After Publishing

Planned support channels:

- GitHub Issues for bugs and feature requests
- Marketplace reviews for public feedback
- Security-sensitive reports should not include private source code, logs,
  secrets, tokens, or proprietary stack traces

## Useful Bug Report Format

```text
VS Code version:
DebugBrief version:
Operating system:
Command used:
Input type: selection / clipboard / file path
Expected result:
Actual result:
Minimal sample log:
```
